# AI-DLC Demo Showcase 요구사항 정의서

## 1. 프로젝트 개요

### 1.1 Intent Analysis
| 항목 | 분석 결과 |
|------|----------|
| **User Request** | AWS Summit 부스용 AI-DLC 자동 데모 쇼케이스 웹앱 |
| **Request Type** | New Project |
| **Scope** | System-wide (프론트엔드 + 백엔드 + AI 연동) |
| **Complexity** | Moderate |

### 1.2 서비스 비전
AWS Summit 2만명 방문객에게 AI-DLC의 가치를 시각적으로 체험하게 하는 인터랙티브 데모 웹앱

### 1.3 핵심 가치 제안
- 방문객이 "나 뭐뭐뭐 만들고 싶어"라고 입력하면 AI-DLC 전체 과정(Inception → Construction → Operations)이 자동으로 진행
- 실제 AI가 동적으로 생성하되, 타이핑 효과로 천천히 화면에 표시
- 질문은 최소화하고 General한 선택으로 자동 응답하여 빠른 데모 진행
- 최종적으로 실제 동작하는 MVP UI 컴포넌트 + AWS 아키텍처 다이어그램 표시
- Production 예상 일정/인력 정보 제공으로 AI-DLC의 효율성 강조

## 3. 핵심 기능 요구사항

### 3.1 시작 화면
**목적**: 방문객이 프로젝트 아이디어를 입력하는 진입점

**기능 요구사항**:
- AI-DLC 브랜딩 및 간단한 소개 문구
- 텍스트 입력 필드: "어떤 서비스를 만들고 싶으세요?"
- "시작하기" 버튼
- 예시 프로젝트 제안 (클릭 시 자동 입력)

### 3.2 AI-DLC 전체 과정 자동 진행 화면
**목적**: AI-DLC 전체 라이프사이클을 Kiro IDE 환경에서 시각적으로 보여주기

**기능 요구사항**:
- **Kiro IDE 레이아웃 시뮬레이션**:
  - 좌측: 파일 탐색기 (aidlc-docs 폴더 구조)
  - 중앙: 에디터 영역 (문서/코드 표시)
  - 하단: 터미널/출력 패널
  - 상단: 탭 바 (열린 파일들)
- **마우스 포인터 애니메이션**:
  - 가상 마우스 커서가 화면에 표시
  - 파일 탐색기에서 파일 클릭하는 동작 시뮬레이션
  - 버튼 클릭, 메뉴 선택 등 모든 동작을 마우스로 표현
  - 부드러운 이동 애니메이션 (ease-in-out)
- **실제 AI(LLM) 호출**: 사용자 입력 기반 동적 생성
- **타이핑 효과**: AI 응답을 천천히 화면에 표시 (스트리밍)
- **자동 진행**: 일정 속도로 계속 진행 (사용자 개입 불필요)
- **AI-DLC 전체 단계 시각화**:
  - **INCEPTION PHASE**:
    - Requirements Analysis (핵심 요구사항 도출)
    - Application Design (컴포넌트 구조)
  - **CONSTRUCTION PHASE**:
    - Code Generation (실제 코드 생성)
  - **OPERATIONS PHASE**:
    - Infrastructure Design (AWS 아키텍처 설계)
    - Deployment Plan (배포 계획)
- **질문 최소화**: 
  - 핵심 질문 1-2개만 자동 생성
  - General한 선택으로 자동 응답 (사용자 개입 없음)
  - 데모 목적에 맞게 빠른 진행 우선

**데모 시나리오 예시** (초등학생도 이해 가능):
1. 마우스가 채팅 입력창으로 이동
2. "할일 관리 앱 만들어줘" 타이핑
3. 마우스가 전송 버튼 클릭
4. 좌측 파일 탐색기에 `aidlc-docs/` 폴더 생성됨
5. 마우스가 `requirements.md` 파일 클릭
6. 에디터에 요구사항이 타이핑되며 작성됨
7. 마우스가 다음 파일들을 순차적으로 클릭하며 진행
8. 코드 파일 생성 시 실제 코드가 타이핑됨
9. AWS 아키텍처 다이어그램 표시
10. MVP 미리보기 화면으로 전환

**UI/UX 요구사항**:
- Kiro IDE 다크 테마 스타일
- 현재 Phase/Stage 인디케이터 (INCEPTION → CONSTRUCTION → OPERATIONS)
- 진행률 표시 (프로그레스 바)
- 마우스 포인터: 부드러운 이동 + 클릭 시 시각적 피드백
- 파일 생성 시 파일 탐색기 애니메이션
- 초등학생도 따라갈 수 있는 천천히 진행되는 속도

### 3.3 MVP 화면 미리보기
**목적**: 생성된 프로젝트의 실제 동작하는 UI + AWS 아키텍처 표시

**기능 요구사항**:
- **실제 동작하는 React 컴포넌트** 렌더링
- AI가 생성한 간단한 UI 컴포넌트 표시
- 인터랙티브 요소 (버튼 클릭, 입력 등) 동작
- **AWS 아키텍처 다이어그램** 표시:
  - 프로젝트에 맞는 AWS 서비스 구성도
  - 주요 서비스 아이콘 및 연결 관계
  - 예: API Gateway → Lambda → DynamoDB 등

**UI/UX 요구사항**:
- 탭 또는 토글로 MVP UI / Architecture 전환
- 디바이스 프레임 (모바일/데스크톱) 내 MVP 미리보기
- AWS 아키텍처는 다이어그램 형태로 시각화
- "이것이 당신의 MVP입니다" 안내 문구

### 3.4 결과 요약 화면
**목적**: AI-DLC의 효율성을 수치로 보여주기

**기능 요구사항**:
- **소요 시간 표시**: "당신은 방금 X분 만에 MVP를 구현했습니다"
- **기존 방식 비교**: "기존 방식으로는 약 Y일이 소요됩니다"
- **Production 예상 정보** (AI 동적 추정):
  - 필요 직군 구성 (예: 프론트엔드 2명, 백엔드 1명, 디자이너 1명)
  - 예상 소요 기간 (예: 약 3주)
  - 예상 인력 (예: 4명)
- **QR 코드**: AI-DLC 관련 자료 링크 제공
- **"처음으로 돌아가기" 버튼**: 클릭 시 시작 화면으로 이동

### 3.5 멀티 세션 지원
**목적**: 여러 디바이스에서 동시 데모 가능

**기능 요구사항**:
- 각 세션 독립적으로 동작
- 세션 간 상태 격리
- 동시 접속자 처리

### 3.6 사용자 입력 로깅 (백엔드)
**목적**: 방문객들이 어떤 프로젝트 아이디어를 입력했는지 분석용 데이터 수집

**기능 요구사항**:
- **로그 저장 항목**:
  - 타임스탬프 (ISO 8601)
  - 세션 ID
  - 사용자 입력 텍스트 (프로젝트 아이디어)
  - 데모 완료 여부
  - 소요 시간
- **저장 방식**: 서버 측 로그 파일 또는 간단한 DB
- **관리자 조회**: 로그 데이터 조회용 간단한 API 또는 파일 다운로드

**데이터 활용**:
- AWS Summit 후 방문객 관심사 분석
- 인기 있는 프로젝트 유형 파악
- 데모 완료율 통계
