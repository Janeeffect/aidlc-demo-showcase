## CR-01. 산업별 예시 시나리오 콘텐츠 분리

**현상:**
- 첫 화면(Screen 1)에서 서로 다른 산업별 예시 시나리오를 클릭해도, 두 번째 화면(Screen 2)에서 진행되는 시나리오 내용이 모두 유사하거나 동일함

**요구사항:**
- 각 산업별 예시 시나리오를 클릭했을 때, Screen 2에서 **해당 산업에 맞는 고유한 시나리오 콘텐츠**가 표시되어야 함
- 산업 A ≠ 산업 B ≠ 산업 C → 각각 **구별되는 시나리오 흐름과 내용**으로 진행

**확인 필요 사항:**
- 각 산업별 시나리오 콘텐츠를 사람이 직접 작성하여 제공할지, AI가 자동 생성하도록 할지 결정 필요

## CR-02. 마우스 커서 시각적 개선

**현상:**
- 현재 마우스 포인터(커서)가 흰색으로 표시되어 화면에서 거의 보이지 않음

**요구사항:**
- 커서 스타일 변경:
  - **형태:** 큰 원형(동그라미)
  - **색상:** 노란색 (`yellow` 계열)
  - **목적:** 사용자가 커서 위치를 쉽게 인식할 수 있도록 가시성 확보

## CR-03. MD 파일 탐색기 - 파일별 콘텐츠 연동 오류 수정

**현상:**
- Screen 2의 MD 파일 탐색기에서 어떤 파일을 클릭해도, **항상 가장 최근에 생성된 MD 파일의 내용만** 표시됨
- 파일 선택에 따른 콘텐츠 전환이 동작하지 않음

**요구사항:**
- 파일 탐색기에서 **특정 MD 파일을 클릭하면, 해당 파일의 고유 내용이 표시**되어야 함
- `파일A.md` 클릭 → 파일A 내용 표시
- `파일B.md` 클릭 → 파일B 내용 표시
- 선택된 파일과 표시되는 콘텐츠가 **1:1로 정확히 매핑**되어야 함

---

## CR-04. 결과 화면(Screen 3) 탭별 콘텐츠 분리

**현상:**
- Screen 3(결과 화면)에서 AWS 아키텍처, 비즈니스 로직 등 **모든 탭의 결과 내용이 어떤 시나리오를 선택해도 동일하게 표시**됨
- Screen 1에서 서로 다른 예시를 선택해도 최종 결과가 같음

**요구사항:**
- Screen 1에서 선택한 산업/시나리오에 따라 Screen 3의 **모든 탭(AWS 아키텍처, 비즈니스 로직 등)의 결과가 해당 시나리오에 맞게 다르게 표시**되어야 함
- 시나리오 선택 → 중간 과정 → 최종 결과까지 **End-to-End로 콘텐츠가 일관되게 분리**되어야 함

## CR-05. 탐색기 패널 - MD 파일 선택 시 콘텐츠 미변경 (재발/미해결)

**화면:** Screen 2 (두 번째 페이지) → 좌측 "탐색기" 패널

**현상:**
- 탐색기 패널에서 어떤 MD 파일을 클릭해도 **동일한 내용만 표시**됨
- 파일 선택에 따른 콘텐츠 전환이 전혀 동작하지 않음

**요구사항:**
- 각 MD 파일 클릭 시, **해당 파일의 고유 콘텐츠로 동적 변경**되어야 함
- 파일명과 표시 콘텐츠가 **1:1 매핑**되어야 함

**우선순위:** 높음

## CR-07. 단계별 채팅 대화 깊이 강화 (MVP → Product-Ready)

**화면:** Screen 2 (두 번째 페이지) → 우측 채팅 영역

**현상:**
- 각 단계(Step)별 AI 채팅 대화가 **너무 짧고 얕음**
- 실제 AI-DLC(AI Development Life Cycle)를 수행하는 수준이 아닌, 단순 데모 수준에 머물러 있음

**요구사항:**
- 각 단계별 채팅이 **실제 AI-DLC를 진행하는 것처럼 깊이 있는 대화**로 구성되어야 함
- **MVP 수준이 아닌 Product-Ready 수준**의 대화 흐름 필요
  - 요구사항 구체화, 엣지케이스 논의, 기술 스택 선정 근거, 트레이드오프 검토 등 포함
  - 단순 "~하겠습니다" 응답이 아닌, **질문-확인-반영의 다회전 대화**
- 대화 턴 수 및 내용의 밀도를 **대폭 증가**

**우선순위:** 높음

---

## CR-08. 결과 화면 산출물 품질 강화 (MVP → Product-Ready)

**화면:** Screen 3 (마지막 페이지) → 전체 탭 (AWS 아키텍처, 비즈니스 로직 등)

**현상:**
- 최종 결과물이 **MVP 수준의 간략한 내용**에 머물러 있음
- 실제 프로덕션에 적용하기에는 깊이가 부족

**요구사항:**
- 모든 결과 탭의 산출물을 **Product-Ready 수준으로 한 단계 깊이 있게** 구현
  - AWS 아키텍처: 단순 구성도가 아닌 **보안, 스케일링, 모니터링, 비용 최적화** 등 포함
  - 비즈니스 로직: 단순 흐름이 아닌 **예외 처리, 유효성 검증, 에러 핸들링** 등 포함
  - 기타 탭도 동일하게 **실제 구현 가능한 수준의 상세도** 확보

**우선순위:** 높음

## CR-09. 채팅 안내 문구 내 버튼 위치 안내 오류 수정

**화면:** Screen 2 (두 번째 페이지) → 우측 채팅 영역

**현상:**
- 채팅 완료 시 `"완료되었습니다. 아래 <다음> 버튼을 눌러주세요"` 라고 안내하지만, **실제 <다음> 버튼은 우측 상단에 위치**함
- 사용자가 버튼을 찾지 못하는 UX 혼선 발생

**요구사항:**
- 안내 문구를 **실제 버튼 위치에 맞게 수정**
  - 수정 전: `"아래 <다음> 버튼을 눌러주세요"`
  - 수정 후: `"우측 상단의 <다음> 버튼을 눌러주세요"` (또는 실제 위치에 맞는 표현)

**우선순위:** 낮음 (단순 텍스트 수정)

## CR-10. 초기 화면 로딩 속도 이슈

**화면:** Screen 1 (첫 화면)

**현상:**
- 첫 화면 로딩이 **비정상적으로 오래 걸림**

**요구사항:**
- **원인 파악 우선 필요**
  - 가능 원인 체크리스트:
    - [ ] 대용량 에셋(이미지/영상/폰트) 미최적화
    - [ ] 불필요한 API 호출 또는 동기 블로킹
    - [ ] 번들 사이즈 과대
    - [ ] 외부 리소스(CDN, 폰트 등) 로딩 지연
    - [ ] 컴포넌트 초기 렌더링 시 과도한 연산
- 원인 파악 후 **로딩 시간 단축 조치** 수행
- 원인과 조치 결과를 **리포트로 공유**

**우선순위:** 높음
